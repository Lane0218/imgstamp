# ImgStamp UI 设计文档

## 0. 现有界面问题诊断（基于截图与目标风格）
**目标风格：轻量克制的工具感/工程感（类似 Windows 生产力工具）**
- **层级不清**：窗口背景、面板、预览画布亮度接近，主区不突出，整体像“白纸铺满屏”。
- **密度失衡**：预览区留白偏大、缩略图间距偏厚，信息密度不一致，显得“空、散”。
- **控件不一致**：圆角、阴影、边框粗细混用，像默认组件拼接，缺乏统一语言。
- **状态不明确**：缩略图状态点过小且语义不清，已选/待完善/完成不易区分。
- **排版层级弱**：标题/字段/内容区分不足，右侧面板读起来像一段连续文本。
- **主动作不突出**：导出/复制等高频操作与普通按钮权重接近，缺少明显主操作锚点。

## 1. 设计原则
- **工程导向**：优先展示信息密度和操作效率，不做过度装饰。
- **浅色单一主题**：只提供浅色主题，保证清晰与易读。
- **工具感/工程感**：界面应像生产力工具而非相册/社交产品。
- **直观反馈**：所有状态变化（选中、修改、保存）都有即时、可感知的反馈。
- **符合直觉**：遵循 Windows 常见桌面软件布局（如资源管理器、Lightroom 简化版）。

## 2. 视觉风格与设计基准
### 2.1 风格关键词
- **轻量、克制、清晰、可扫视**
- 明确边界与层级，避免装饰性渐变和高饱和配色。

### 2.2 色板（浅色主题）
- 背景：`#F6F7F9`
- 面板：`#FFFFFF`
- 画布：`#F0F2F5`
- 状态栏背景：`#EEF2F7`
- 边框：`#E5E7EB`
- 主文字：`#0F172A`
- 次文字：`#475569`
- 禁用文字：`#94A3B8`
- 主色：`#2563EB`
- 成功：`#16A34A`
- 警告：`#F59E0B`
- 错误：`#DC2626`

### 2.3 字体
- **仅系统字体**：优先中文显示  
  - Windows 推荐：`Segoe UI`, `Microsoft YaHei`, `PingFang SC`（在 Windows 上以 `Segoe UI` + `Microsoft YaHei` 为主）
- 不引入在线字体，避免加载不确定性。

### 2.4 字号与行高（密度适中）
- 标题/分组标题：16–18px，字重 600
- 正文：13–14px，字重 400–500
- 辅助/提示：12px，字重 400
- 行高：1.4

### 2.5 间距与栅格
- 采用 8pt 间距体系：`4/8/12/16/24/32`
- 组件内边距：12–16px
- 主要区域边距：16–24px

### 2.6 圆角、阴影、边框
- 圆角：卡片 8px，按钮 6px，输入框 6px
- 阴影：`0 1px 8px rgba(0,0,0,0.06)`（轻量即可）
- 边框：1px，使用 `#E5E7EB`

### 2.7 交互与状态规范
- Hover：颜色变化 6–10%，过渡 150–200ms
- Focus：2px 外描边，主色 `#2563EB`（可用 box-shadow 表达）
- Disabled：文字与边框统一变浅，按钮不可点
- Error：控件描边变红，提示文本紧邻表单
- 选中：2px 主色描边 + 轻微背景填充（1–2%）

### 2.8 图标规范
- 使用 `@ant-design/icons`，保持 16px/18px/20px 三级尺寸
- 图标与文字之间间距 6–8px
- 不使用 Emoji 作为功能图标

### 2.9 层级与密度策略
- **三层背景**：窗口背景 `#F6F7F9`、面板 `#FFFFFF`、画布 `#F0F2F5`，通过明度差区分主次。
- **阴影克制**：仅预览画布与关键卡片使用轻阴影，其余以边框分隔为主。
- **密度统一**：缩略图区/表单区/预览区使用一致的 8pt 间距体系，避免“空与挤”并存。
- **主操作显著**：导出为主按钮，其余为次按钮或文本按钮。

## 3. 启动器窗口（项目入口）
应用启动后首先进入独立的 **启动器窗口**，在此选择新建/打开项目，完成后再进入主窗口。

### 3.1 启动器尺寸与布局
- **窗口建议尺寸**：`900 x 560`，最小 `820 x 520`。
- **布局结构**：左侧“最近项目列表” + 右侧“操作区”。
- **风格参考**：Obsidian 的入口布局（左列列表、右侧大操作卡片），但保持浅色与工具感。

### 3.2 最近项目列表（左侧）
- 展示最近打开项目，单项显示 **项目名 + 路径**。
- 点击条目直接打开主窗口并关闭启动器。
- 列表为空时展示提示文案：“暂无最近项目”。

### 3.3 操作区（右侧）
#### 3.3.1 入口页
- 主按钮：`新建项目` / `打开项目`（并排、等权、方形带图标，尺寸与卡片高度协调）
- 辅助提示：`拖入项目文件（.json）开始`

#### 3.3.2 创建项目页（同窗口内切换）
- 顶部返回入口：`返回`
- 表单字段：
  - **项目名称**（必填）
  - **图片文件夹**（必选，按钮“浏览”）
  - **项目文件**（必选，选择 `.json` 保存位置）
- 操作按钮：`创建`

### 3.4 交互规则
- 选择完成后 **主窗口打开，启动器关闭**。
- 拖拽导入 **只在启动器支持**，仅接受 `.json` 项目文件。
- 主窗口为单实例。

## 4. 主窗口布局
采用 **“顶部菜单栏 + 三栏内容区 + 底部状态栏”** 的布局。

### 4.1 布局约束与尺寸
- **窗口最小尺寸**：建议 `1100 x 720`，确保三栏都可用。
- **默认比例**：左 / 中 / 右 ≈ `22% / 53% / 25%`。
- **最小宽度**：
  - 左栏 ≥ 240px（缩略图至少 2 列可显示）
  - 中栏 ≥ 520px（预览不被压扁）
  - 右栏 ≥ 320px（表单不挤压）
- **可拖拽分隔条**：三栏之间允许拖拽调宽，拖拽时保持最小宽度限制。
- **顶部/底部高度**：
  - 顶部菜单栏：44px
  - 底部状态栏：28px
- **滚动规则**：
  - 左栏列表独立滚动
  - 中栏画布固定居中，不跟随左/右滚动
  - 右栏表单独立滚动

```
+-----------------------------------------------------------------------------------+
|  [顶部菜单栏]                                                                     |
+---------------------+-------------------------------+-----------------------------+
|                     |                               |                             |
|  [左侧：资源管理器]  |      [中间：实时预览区]        |    [右侧：属性编辑面板]      |
|  显示图片缩略图列表  |      显示当前选中图的成品效果   |    编辑当前图的元数据        |
|  占据 20% 宽度       |      占据 55% 宽度            |    占据 25% 宽度            |
|                     |                               |                             |
+---------------------+-------------------------------+-----------------------------+
|  [底部状态栏]                                                                     |
+-----------------------------------------------------------------------------------+
```

## 5. 详细区域设计

### 5.1 顶部菜单栏
仅保留 **“项目 / 帮助”** 两个菜单入口，保持简洁工具感。
- **项目**：
  - `新建项目`（打开图片文件夹）
  - `打开项目`
  - `保存项目`
  - `退出`
- **帮助**：
  - `关于`
  - `快捷键`

### 5.2 左侧：资源管理器
以垂直滚动列表形式展示源图片。
- **布局**：自适应网格（根据面板宽度自动调整列数，默认 2 列，最小 2 列、最大 4 列）。
- **卡片内容**：
  - 小缩略图（等高，保持原图比例，可能出现留白）。
  - 状态标识（缩略图右上角小角标或圆点，建议 8–10px）：
    - 警告色：已选但信息不完整。
    - 成功色：已选且信息完整。
- **间距与圆角**：卡片圆角 8px，网格间距 8–12px，保持紧凑可扫视。
- **选中样式**：2px 主色描边 + 轻微背景填充，确保一眼可见。
- **状态图例**：在左栏顶部提供小型图例（完成/待完善），降低理解成本。
- **交互**：
  - **单击**：切换中间预览区显示的图片，并联动右侧面板。
  - **Ctrl/Shift + 点击**：多选（支持批量编辑，中间预览区显示该批次的第一张图）。
  - **右键菜单**：`移除`、`打开所在文件夹`。
  - **分页**：当图片数量过多时分页显示；每页数量随窗口高度自适应，确保当前页缩略图完整显示。
  - **不展示文件名**：缩略图不显示/不悬停显示文件名，保持干净列表。

### 5.3 中间：实时预览区
**核心体验区域**，所见即所得。
- **画布**：
  - 背景色：浅灰 `#F0F2F5`，可加极弱噪点或 2% 渐变以增强层次。
  - 画布外框：1px 边框 + 轻阴影，提升“成品感”。
  - 外边距：画布与容器保持 24–32px 留白，避免过空或过挤。
  - **实时渲染**：显示当前照片叠加了白边、日期、地点、描述后的最终效果。
  - **渲染逻辑**：
    - 默认生成预览图，使用主进程 Sharp 渲染缩小版（与导出同一布局算法）。
    - 当右侧数据变更时，防抖触发预览更新。
  - **标题栏右侧操作区**：
    - **导出尺寸下拉**（5/大5/6/大6）
    - **导出成品**（主按钮，强调主流程）
    - 图标按钮：放大 / 缩小 / 查看原图
  - **多选模式**：预览区显示本次多选中的第一张图片。

### 5.4 右侧：属性编辑面板
用于编辑当前选中图片的信息。若多选，则显示“批量编辑模式”。

#### 5.4.1 头部信息
- 显示当前文件名 / "已选中 N 张图片"。
- 标题栏右侧放置图标按钮：`上一张` / `下一张`。
- 说明：选中即加入导出列表，不提供额外开关。

#### 5.4.2 核心表单
> 输入控件高度统一 32–34px，标签左对齐。
1.  **拍摄日期**
    - 组件：`DatePicker` (仅日期)。
    - 辅助按钮：`复位到 EXIF`（紧贴日期输入框右侧，可用图标表达）。
2.  **拍摄地点**
    - 组件：`AutoComplete` (自动补全输入框)。
    - 数据源：从历史输入记录中读取。
    - 占位符："例如：上海市"。
    - 默认值："湖南长沙"。
3.  **描述**
    - 组件：`Input` (单行文本框)。
    - 占位符："记录当下的心情或事件（限单行）..."。

#### 5.4.3 快捷操作区
位于面板底部。
- 按钮：`应用日期` / `应用地点` / `应用描述`
- 按钮：`应用全部`
  - 单选时显示但禁用，多选时可用。
  - 视觉上为整行主按钮。
  - 规则：点击后将对应字段直接覆盖所有选中照片。
> 快捷区可固定在面板底部，与表单区有分割线，主按钮视觉权重略高。

### 5.5 底部状态栏
- 样式：背景 `#EEF2F7`，顶部 1px 分割线，文字使用次文字色。
- 左侧：`总计: 200 张` | `已选: 15 张` | `待完善: 5 张`
- 右侧：显示当前处理状态（如 "正在读取 EXIF..." 或 "已保存"）。

### 5.6 导出弹窗
点击“导出成品”后弹出。
- **设置项**：
  - **输出目录**：选择文件夹。
  - **图片质量**：`高质量 (JPG 95%)` / `中等`。
- **预览区**：默认生成并显示第一张图的效果预览。
- **操作**：`开始导出` (显示进度条)。

## 6. 交互与反馈细节

- **快捷键支持**：
  - `← / →`：切换选中图片。
  - `Space`：标记/取消标记“已选”。
  - `Ctrl + Enter`：开始导出。
  - `Ctrl + S`：保存项目。
- **加载状态**：
  - 图片加载时显示骨架屏或加载转圈。

## 7. 组件库选型
- **布局**: `Layout`, `Sider`, `Content`, `Header`, `Footer`
- **输入**: `DatePicker`, `Input`, `AutoComplete`, `Switch`
- **反馈**: `Message` (全局提示), `Modal` (导出/大图), `Spin` (加载)
- **图标**: `@ant-design/icons` (使用通用图标如 `FolderOpenOutlined`, `SaveOutlined`)
